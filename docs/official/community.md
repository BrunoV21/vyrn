# Community

vyrn is early Rust package infrastructure. Feedback should stay grounded in the current PRD scope: terminal-native, token-efficient, model-agnostic, and local-first.

## Useful links

- [Bug reports](https://github.com/BrunoV21/vyrn/issues/new)
- [Feature requests](https://github.com/BrunoV21/vyrn/issues/new)
- [Open issues](https://github.com/BrunoV21/vyrn/issues)
- [Repository](https://github.com/BrunoV21/vyrn)

## Good issues include

- The local model or endpoint used.
- Context budget and approximate session length.
- Token stats from the affected turn.
- Relevant `.vyrn/config.toml`, `.vyrn/models.toml`, or `.mcp.json` snippets with secrets removed.
